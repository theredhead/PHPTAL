PHPTAL

These attributes are not defined in TAL specifications but should make the like of template designers easier for some kind of jobs. I made aliases to these attributes so you can use tal:include or tal:src-include instead of phptal:include and phptal:src-include.

phptal:include

This attribute include and evaluate an external template in the current one. The included template inherit from current template context (variables). 

<span phptal:include="string:path/to/some/file.html"/>

phptal:src-include

This attribute include the content of a file without interpreting it. 

<pre phptal:src-include="string:somepage.html">
here is included the content of somepage.html 
</pre>

If you attempt to use this attribute with not well formated source, you will break your XML/XHTML page format, phptal:src-include assume the page you try to include is a well formatted XML file.

If you just want to include the source of an unknown format file (programming language source for instance), you can do as follow : 

<!-- first define a variable which receive the file content --> 
<span tal:define="myvar"> 
  <span phptal:src-include="string:somepath.html"/> 
</span>

<!-- then echoing the variable, with htmlencode it --> 
<pre tal:content="myvar"/>
